# 인터뷰 준비: collar 방향성 결정

**목적:** "말 잘 듣는 AI"를 모든 프로젝트에서 가능하게 하는 시스템의 방향성을 함께 결정

---

## 현황 요약

당신이 현재 가진 것:
- **OMC** (oh-my-claude-code): 글로벌 Claude Code 하네스 (`~/.claude/`)
- **Paperclip**: AI 에이전트 회사 제어 플레인 (`~/Documents/dev/ai/paperCompany/paperclip/`)
- **gstack**: 스킬 시스템 (ship, qa, review, investigate 등)
- **58개 이상의 AI 프로젝트**: 각각 다른 하네스 설정 또는 무설정

핵심 문제:
> 프로젝트마다 새로 하네스를 짜야 하거나, 아예 없어서 AI가 제멋대로 동작한다.

---

## 배경 개념 (분석한 영상들에서)

### 하네스 엔지니어링 4요소 (Dori 영상)
1. **헌법** - 우리 팀은 이렇게 일한다 (CLAUDE.md)
2. **작업 구조** - 무엇을 어떻게 만들지 (task spec, PRP)
3. **검증** - 결과가 맞는지 체크 (typecheck, tests)
4. **실행 루프** - 수정→검증→반복 (hooks, automation)

### OMX (oh-my-codex)의 핵심 (허예찬 발표)
- OMC vs OMX vs OpenCode: 런타임이 다르다
- 하네스로 1,000줄짜리 코드도 사람 없이 가능
- 에이전트들이 "팀처럼" 동작하는 것이 핵심

### MIT 연구가 말하는 "말 잘 듣는 AI"의 역설
- 인지 부채: AI에 수동적으로 의존하면 사람이 멍청해짐 
- 역설: 사람이 명확하게 사고해야 AI도 잘 일함
- **결론: 좋은 하네스 = 사람의 사고를 대체하지 않고 보조**

## 인터뷰 질문 (내가 드릴 것)

### 1. 비전 확인
> "모든 프로젝트에서 말 잘 듣는 AI"가 구체적으로 어떤 상태를 말하는 건가요?
> 예) 새 프로젝트 만들 때 즉시 제대로 동작하는 AI? 기존 프로젝트들도 업그레이드?

**예상 답변 옵션:**
- A) 새 프로젝트 시작할 때 collar init 하나로 하네스 완성
- B) 기존 58개 프로젝트에 레트로핏
- C) Paperclip이 자동으로 어느 프로젝트든 관리

---

### 2. 범위 확인
> collar는 Claude Code(OMC) 전용인가요, 아니면 OMX/Cursor/Codex도 포함인가요?

**상황:**
- OMC: 이미 잘 작동 중, 글로벌 하네스 있음
- OMX: 허예찬이 "요즘 메인 드라이버"라고 함, 별도 런타임
- Cursor: 다른 IDE 기반

**내 제안:** Claude Code 우선, 나중에 확장 (이미 있는 인프라 활용)

---

### 3. Paperclip 관계
> collar와 Paperclip은 어떻게 연결될까요?

**현재 Paperclip:**
- AI 에이전트 = 직원 (역할, 예산, 보고 체계)
- 태스크 기반 오케스트레이션
- 이미 Claude Code, Codex, Cursor 어댑터 있음

**옵션:**
- A) collar가 Paperclip 위에서 동작 (Paperclip이 오케스트레이터)
- B) collar가 독립적으로, Paperclip은 나중에
- C) collar가 Paperclip의 플러그인/어댑터로

---

### 4. 우선순위
> 지금 당장 가장 불편한 게 뭔가요?

**선택지:**
- A) 새 프로젝트 시작할 때마다 CLAUDE.md를 처음부터 짜는 게 귀찮다
- B) 기존 프로젝트들의 AI가 제멋대로여서 일관성이 없다
- C) 여러 에이전트가 동시에 돌 때 조율이 안 된다
- D) 비용/토큰 추적이 안 된다

---

### 5. 구현 접근법
> 직접 코드를 짜는 것 vs 있는 것을 조합하는 것?

**현재 있는 것:**
- OMC: 글로벌 하네스 프레임워크 (이미 동작)
- Paperclip: 오케스트레이션 플레인 (이미 동작)
- gstack: 스킬 시스템 (이미 동작)

**내 관찰:** 도구들은 이미 있다. 빠진 건 "연결"과 "표준화"다.

## 내가 제안하는 방향성 (인터뷰 전 생각해 두기)

### collar = 하네스 표준화 + 연결 레이어

```
새 프로젝트
    ↓
collar init (10초)
    ↓
프로젝트 타입 감지 (Node/Python/Rust...)
    ↓
적절한 CLAUDE.md + AGENTS.md 자동 생성
    ↓
Paperclip에 프로젝트 등록 (선택)
    ↓
즉시 하네스 완성
```

**핵심 가치:**
- 0 → 하네스 완성: 10초
- 표준화된 검증 루프: 언어별 자동 설정
- Paperclip 연동: 선택적

**비교:**
| | 지금 | collar 이후 |
|---|---|---|
| 새 프로젝트 CLAUDE.md | 30분~1시간 | 10초 |
| 에이전트 일관성 | 프로젝트마다 다름 | 표준화 |
| 검증 루프 | 수동 | 자동 |
| 비용 추적 | 없음 | Paperclip 연동 |

## 실행 가능한 첫 번째 스텝

인터뷰 완료 후 바로 할 수 있는 것:

1. **collar 프로젝트 초기화** (`~/Documents/dev/ai/collar/`)
2. **CLAUDE.md base 템플릿** 작성
3. **`collar init` 스크립트** 작성 (bash 기반, 빠르게)
4. **첫 번째 프로젝트에 적용** (검증)
