# 용어집 — collar / 하네스 엔지니어링 프로젝트
**날짜:** 2026-05-14  
**범위:** 이 대화 세션에서 사용된 모든 기술 용어

---

## A

**AGENTS.md**  
프로젝트 레포지토리에 체크인하는 에이전트 가이드 문서. CLAUDE.md가 "규칙"이라면 AGENTS.md는 "팀 가이드". 에이전트 팀 구조, 에스컬레이션 프로토콜, 모델 라우팅 등을 정의한다. Codex/Claude Code가 자동으로 읽는다.

**--approval-mode (Gemini CLI)**  
`gemini --approval-mode yolo`: AI가 모든 파일 변경을 자동 승인. `auto_edit`: 파일 편집만 자동 승인, 위험한 명령은 사용자에게 물어봄. `default`: 매번 승인 요청. Claude Code의 Permission 모델보다 세분화됨.

---

## B

**Batch 처리**  
여러 파일을 동시에 처리하는 방식. `codex exec`처럼 독립 프로세스를 활용하면 파일별로 병렬 LLM 호출 가능:
```bash
for f in src/*.ts; do codex exec "add types" < "$f" > "typed/$(basename $f)" & done; wait
```

## C

**Claude Code Max 플랜**  
Anthropic의 Claude Code 최고 구독 플랜. "무제한"이지만 실제로는 시간당 레이트 리밋 있음. 대화형(interactive) 세션 중심 설계. `--print` 모드는 비대화형 파이프라인용.

**Claude Code `--print` / `-p` 모드**  
`claude -p "prompt"` 형태로 실행하는 비대화형 모드. 이 모드에서만 다음 기능 사용 가능:  
- `--fallback-model`: API 과부하 시 자동 다른 모델로 폴백  
- `--max-budget-usd`: 최대 비용 한도 설정  
- `--no-session-persistence`: 세션 저장 안 함  
- 대화형 세션에서는 이 기능들이 작동하지 않음

**CLAUDE.md**  
프로젝트 헌법. Claude Code와 Codex가 프로젝트 시작 시 자동으로 읽는 설정 파일. 프로젝트 규칙, 검증 명령어, 도메인 라우팅 등을 정의한다. collar가 생성하는 핵심 산출물.

**collar**  
이 프로젝트. 프로젝트 수준의 하네스(CLAUDE.md + AGENTS.md)를 10초 만에 설치하는 도구. ez2claude(글로벌)와 Paperclip(오케스트레이션) 사이의 프로젝트 레이어. `collar-init` 하나로 실행.

**collar-init**  
`~/Documents/dev/ai/collar/bin/collar-init` — 실행하면 프로젝트 타입을 자동 감지(nextjs/python/rust/go/generic)하고 CLAUDE.md + AGENTS.md + .claude/settings.json을 생성.

**codex exec**  
`codex exec "prompt"` 형태의 OpenAI Codex CLI 비대화형 모드. stdin → LLM → stdout 파이프라인. 상태 없음, 병렬화 trivial. 허예찬이 말한 "가벼운 프로그램"의 대표 사례. 로컬 설치됨 (v0.121.0).

**codex-cli (oh-my-codex, OMX)**  
OpenAI의 Codex CLI. 허예찬이 이걸 마개조한 것이 OMX(oh-my-codex). 비대화형 파이프라인 우선 설계로 오케스트레이션에 적합.

**Cognitive Debt (인지 부채)**  
MIT 연구에서 발견된 개념. AI를 수동적으로 의존하면 뇌의 신경 연결성이 약화됨. 자신의 사고를 유지하면서 AI를 도구로 사용해야 함께 성장.

**컨텍스트 오염 (Context Pollution)**  
Claude Code 장기 세션에서 이전 시도, 실패, 잘못된 전제가 누적되어 현재 응답 품질을 떨어뜨리는 현상. 세션 압축(context compression)으로 일부 해결되지만 lossy.

**컨텍스트 압축 (Context Compression)**  
Claude Code가 컨텍스트 한도에 가까워질 때 이전 대화를 요약해서 줄이는 자동 프로세스. 요약 과정에서 중요한 세부 내용이 손실될 수 있음.

## D

**--dangerously-skip-permissions**  
Claude Code를 모든 권한 승인 없이 실행하는 플래그. 자율 작업에 필요하지만 이진화 문제 있음 (전체 허용 vs 전체 차단만 가능).

---

## E

**ez2claude**  
사용자의 글로벌 OMC 구현체. `~/.claude/` 에 위치. 모든 프로젝트에 공통으로 적용되는 규칙, 에이전트, 훅 포함. collar와 역할 분담: ez2claude = "어떻게 일할지", collar = "이 프로젝트에서 무엇을 만들지".

---

## F

**--fallback-model**  
`claude --print --fallback-model claude-haiku-4-5-20251001` — Claude API 과부하 시 자동으로 지정한 다른 모델로 전환. **단, `--print` 모드에서만 작동. 대화형 세션에서는 미작동.**

---

## G

**Gemini CLI**  
Google의 Gemini AI CLI. `gemini -p "prompt" --approval-mode yolo`로 비대화형 자동 실행 가능. 특장점: 1M 토큰 컨텍스트(대용량 파일 처리), 멀티모달(이미지+코드). 로컬 설치됨 (v0.36.0).

**gemini-mcp MCP server**  
Claude Code에 연결된 MCP(Model Context Protocol) 서버. Gemini의 기능을 Claude Code 대화 안에서 직접 호출 가능:
- `gemini_vision`: 이미지/스크린샷 분석
- `gemini_code`: 코드 리뷰
- `gemini_summarize`: 긴 텍스트 요약
- `gemini_prompt`: 일반 텍스트 질의

**gstack**  
Claude Code용 skill 레이어. `/gstack` 명령으로 실행. 작업별 전문 워크플로우 제공 (/investigate, /qa, /review, /ship 등). collar가 영감받은 도구 중 하나.

**gstack "Boil the Lake" 원칙**  
"AI가 한계 비용을 0에 가깝게 만들 때, 완전한 것을 해라." — 에이전트는 가능하면 완전한 작업을 해야 한다는 gstack의 철학.

---

## H

**하네스 (Harness)**  
AI가 잘 일할 수 있도록 설계된 작업 환경 전체. CLAUDE.md + AGENTS.md + 검증 명령 + 훅/자동화의 조합. "말 잘 듣는 AI" = 잘 설계된 하네스 위에서 작동하는 AI.

**하네스 엔지니어링 4요소**  
1. 헌법(Constitution): CLAUDE.md, AGENTS.md  
2. 작업구조(Work Structure): task spec, PRP, plan docs  
3. 검증(Verification): typecheck, tests, CI  
4. 실행루프(Execution Loop): hooks, automation, skills

---

## M

**MCP (Model Context Protocol)**  
AI에게 외부 도구/서비스 접근 권한을 부여하는 프로토콜. `~/.claude.json`의 `mcpServers` 설정으로 관리. 현재 설정됨: gemini MCP.

**멀티-LLM 라우팅**  
여러 LLM을 작업 특성에 따라 자동으로 선택하는 설계. 현재 collar 템플릿에 라우팅 테이블로 문서화됨. 실행 자동화는 `collar run` 명령어로 구현 예정.

---

## O

**OMC (oh-my-claude-code)**  
Claude Code용 멀티 에이전트 오케스트레이션 레이어. `~/.claude/`에 위치. 사용자 버전은 v4.9.1 (19 에이전트, 3 보안 훅). ez2claude가 이를 구현한 것.

**OMX (oh-my-codex)**  
허예찬이 OpenAI Codex CLI를 마개조한 별도 런타임. 하네스로 순정과 다른 동작을 구현. codex-cli 기반이므로 비대화형 파이프라인 특성 유지.

---

## P

**Paperclip**  
AI 에이전트 회사 제어 플레인 (오케스트레이션). `~/Documents/dev/ai/paperCompany/paperclip/`. 조직도, 예산, 거버넌스, 하트비트 관리. collar보다 상위 레이어. collar → Paperclip 연동이 장기 목표.

**Permission 이진화 문제**  
Claude Code의 권한 모델 문제. 선택지가 "매번 승인 요청(기본)" vs "전체 허용(--dangerously-skip-permissions)" 두 가지뿐. Gemini의 `--approval-mode auto_edit` 같은 중간 단계가 없음.

**PRP (Product Requirements Prompt)**  
에이전트에게 전달하는 구조화된 작업 지시 문서. Task spec의 발전된 형태.

---

## R

**레이트 리밋 (Rate Limit)**  
Claude Code Max 플랜에 존재하는 시간당 토큰 처리 한도. "무제한"이지만 실제로는 과부하 시 응답 속도가 느려지거나 블로킹됨. 구체적 한도는 Anthropic이 공개하지 않음.

**run_in_background**  
Claude Code에서 에이전트를 백그라운드로 실행하는 파라미터. 병렬 처리에 사용되지만, 각 에이전트 결과가 메인 세션 컨텍스트에 쌓여 점점 무거워지는 단점 있음.

---

## S

**STARTED / TESTED / VERIFIED (완료 3단계)**  
collar v2 템플릿에서 정의한 작업 완료의 세 단계:
- **STARTED**: 코드 변경만 됨, 미검증
- **TESTED**: 로컬 빌드/테스트 통과, UI 확인됨
- **VERIFIED**: 실제 사용자 환경에서 동작 확인됨

AI가 "완료"라고 할 때 어느 단계인지를 명시해야 함.

**Supabase**  
PostgreSQL 기반 오픈소스 BaaS. 인증 + DB + 실시간 구독. socialMakeit의 auth와 DB 레이어로 사용.

## W

**말 잘 듣는 AI**  
이 프로젝트의 핵심 목표. 단순히 명령을 따르는 AI가 아니라 올바른 하네스(환경) 위에서 작동해서 예측 가능하고 신뢰할 수 있는 AI. 하네스를 잘 설계하면 AI가 "말을 잘 듣는" 상태가 됨.
